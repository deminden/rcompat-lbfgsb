# Provenance

`rcompat-lbfgsb` is intended to be an MIT-licensed, clean-room Rust
implementation of the user-visible behavior of R's
`stats::optim(..., method = "L-BFGS-B")`.

Provenance is part of this crate's architecture. Compatibility decisions should
be traceable to public behavior, public documentation, generated fixtures, or
permissively licensed dependencies. They should not be traced to copied,
translated, or paraphrased GPL/R implementation source.

## Clean-room boundary

- No R source code, R C source, R Fortran source, or GPL-derived optimizer code
  may be copied, translated, paraphrased, or used as an implementation source.
- R may be run externally during development to generate input/output parity
  fixtures. The crate must not call, link, shell out to, embed, or depend on R at
  runtime.
- Normal `cargo test` must not require R to be installed.
- Public documentation and black-box behavior are acceptable references.
- If upstream optimizer repositories are inspected, store them under the ignored
  `external/` directory.

## Inspection-only upstream material

For license review or behavioral background, a maintainer may clone or copy
upstream repositories into `external/`. This material is not part of the crate,
must not be packaged, and must not be used as source text for Rust
implementation.

The repository at <https://github.com/jonathanschilling/L-BFGS-B> may be useful
for inspection because it collects L-BFGS-B source history, but code from it
must not be copied directly into this crate.

R source snapshots or tarballs may also be kept under `external/` for
inspection-only behavior archaeology. Any compatibility change informed by such
inspection must still be validated through public behavior, generated R traces,
or parity fixtures, and no R source text may be copied, translated, or
paraphrased into the Rust implementation.

## Accepted implementation inputs

- Public R documentation for `optim` and its controls.
- Parity and trace fixtures generated by running R externally during
  development.
- Small committed fixture subsets copied from ignored local real-data fixtures
  when their source, selection rule, and generated optimizer behavior are
  documented.
- Independently written Rust code in this repository.
- Permissively licensed dependencies after their licenses have been verified.

## Real-data fixture policy

`data/` is intentionally ignored and excluded from packaging. Small, reviewed
subsets may be copied into `fixtures/` when they are needed for compatibility
tests and are not implementation source.

The current `fixtures/deseq_real_subset/` data was selected from `data/` by
`scripts/generate_r_fixtures.R`. It contains a compact set of numeric DESeq
workflow outputs and R `optim` results for an independently written standard
negative-binomial GLM objective. It does not contain copied DESeq2, R, C, or
Fortran implementation code.

The current `fixtures/deseq_hard_real_subset/` data was selected from the
ignored `data/lbfgsb_hard_real_2026-06-01/` bundle by
`scripts/generate_hard_real_fixtures.R`. It keeps six hard numeric optimizer
cases per completed contrast and stores generated R `optim` outputs for an
independently written negative-binomial GLM objective using DESeq2's log2-scale
coefficient convention. The generator replays objective-only
`optim(method = "L-BFGS-B")`, matching DESeq2's GLM fallback optimizer shape.
The normal test uses these fixtures as a compact behavioral guard: exact
optimizer-count parity is required for most cases, while weakly identified
directions are judged by objective equivalence.

The ignored `data/lbfgsb_hard_real_2026-06-01/` bundle is also used for local
trace investigation through `scripts/trace_hard_real_case.R`. That helper
invokes R's own `optim(method = "L-BFGS-B")` on the independent objective and
can enable R optimizer trace output with `HARD_REAL_OPTIM_TRACE`; these traces
are inspection artifacts only and are not copied into the implementation.
Current local DESeq2 reference checks should run from
`/home/den/miniforge3/envs/rsfgsea-r460`, verified on 2026-06-06 with R 4.6.0,
Bioconductor 3.23, and DESeq2 1.52.0, so fixture regeneration and trace probes
do not accidentally use the system R installation.

The R `optim` fixture set also includes small raw objective-call trace fixtures
generated by `scripts/generate_r_fixtures.R`. These traces record parameter
vectors passed to an independent test objective and are used to guard
user-visible wrapper behavior such as finite-difference stencil order near
bounds.

The `line_search_refresh_sin_quad` fixture is a synthetic black-box R behavior
case generated by the same script. Its objective is an independently written
finite-box sinusoidal quadratic. Running R `optim(method = "L-BFGS-B")` with
high trace verbosity shows a line-search memory refresh before successful
`factr` convergence; the committed fixture stores only the generated inputs and
public optimizer result, not R implementation text.

The `pgtol_near_upper_uses_bound_distance` fixture is another synthetic
black-box R behavior case generated from an independent one-dimensional linear
objective. It records R's projected-gradient stopping norm near a finite bound:
the raw gradient is large, but the projected component is capped by the
remaining distance to the upper bound, so R reports initial `pgtol`
convergence.

The objective-only finite-box exact Cauchy/subspace free-variable
classification and tolerant supplied-gradient/fallback direction projection are
guarded by generated R fixtures and by the compact hard-real DESeq2 replay
subset. This split was validated through optimizer outputs and counts, not
copied R implementation text.

The smaller objective-only finite-box exact-bound full-step target copy is
likewise a black-box parity rule. It was kept because generated hard-real DESeq2
optimizer replays improved from 35 to 36 exact count matches while preserving
the existing max objective-error and max count-drift envelope; supplied-gradient
cases and larger full-design objective-only cases stay on recomputed `x + d`
arithmetic until their compact subspace path is matched more closely.

The compact subspace minimizer for smaller objective-only finite boxes is an
independent Rust implementation of the public L-BFGS-B compact-matrix
algorithmic formulas. It is validated by generated R optimizer fixtures and
kept behind a dimension gate because the committed hard-real subset shows a
clear worst-count-drift improvement there without relying on copied R, C, or
Fortran source text.

## Dependency policy

Runtime dependencies must be permissively licensed and compatible with this
crate's MIT license target. If an optimizer backend is added, it must be isolated
behind the crate's backend abstraction so that R-compatible semantics remain in
this wrapper layer.

License checks and fixture provenance should be updated whenever a new backend,
dependency, or fixture source is added.
