# Provenance

`rcompat-lbfgsb` is intended to be an MIT-licensed, clean-room Rust
implementation of the user-visible behavior of R's
`stats::optim(..., method = "L-BFGS-B")`.

Provenance is part of this crate's architecture. Compatibility decisions should
be traceable to public behavior, public documentation, generated fixtures, or
permissively licensed dependencies. They should not be traced to copied,
translated, or paraphrased GPL/R implementation source.

## Clean-room boundary

- No R source code, R C source, R Fortran source, or GPL-derived optimizer code
  may be copied, translated, paraphrased, or used as an implementation source.
- R may be run externally during development to generate input/output parity
  fixtures. The crate must not call, link, shell out to, embed, or depend on R at
  runtime.
- Normal `cargo test` must not require R to be installed.
- Public documentation and black-box behavior are acceptable references.
- If upstream optimizer repositories are inspected, store them under the ignored
  `external/` directory.

## Inspection-only upstream material

For license review or behavioral background, a maintainer may clone or copy
upstream repositories into `external/`. This material is not part of the crate,
must not be packaged, and must not be used as source text for Rust
implementation.

The repository at <https://github.com/jonathanschilling/L-BFGS-B> may be useful
for inspection because it collects L-BFGS-B source history, but code from it
must not be copied directly into this crate.

## Accepted implementation inputs

- Public R documentation for `optim` and its controls.
- Parity and trace fixtures generated by running R externally during
  development.
- Small committed fixture subsets copied from ignored local real-data fixtures
  when their source, selection rule, and generated optimizer behavior are
  documented.
- Independently written Rust code in this repository.
- Permissively licensed dependencies after their licenses have been verified.

## Real-data fixture policy

`data/` is intentionally ignored and excluded from packaging. Small, reviewed
subsets may be copied into `fixtures/` when they are needed for compatibility
tests and are not implementation source.

The current `fixtures/deseq_real_subset/` data was selected from `data/` by
`scripts/generate_r_fixtures.R`. It contains a compact set of numeric DESeq
workflow outputs and R `optim` results for an independently written standard
negative-binomial GLM objective. It does not contain copied DESeq2, R, C, or
Fortran implementation code.

## Dependency policy

Runtime dependencies must be permissively licensed and compatible with this
crate's MIT license target. If an optimizer backend is added, it must be isolated
behind the crate's backend abstraction so that R-compatible semantics remain in
this wrapper layer.

License checks and fixture provenance should be updated whenever a new backend,
dependency, or fixture source is added.
